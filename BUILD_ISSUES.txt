ISSUE #1 — PyInstaller crashes with: IndexError: tuple index out of range
────────────────────────────────────────────────────────────────────────────────

Error location:
  File "...modulegraph\util.py", line 13, in iterate_instructions
  IndexError: tuple index out of range

Root cause:
  Python 3.10.0 (first release) has a broken bytecode format.
  PyInstaller 6.x uses dis.get_instructions() to scan all bytecode,
  and Python 3.10.0 produces malformed constant tuples in certain cases.

Fix:
  Upgrade Python to 3.10.6 or higher (recommended: 3.11 or 3.12).

  1. Download from: https://python.org/downloads
     → Tick "Add Python to PATH" during installation.

  2. Delete the old virtual environment:
       rmdir /s /q .venv          (Windows CMD)
       rm -rf .venv               (Linux / macOS)

  3. Create a new venv with the new Python and reinstall:
       python -m venv .venv
       .venv\Scripts\activate.bat         (Windows CMD)
       source .venv/bin/activate          (Linux / macOS)
       pip install -r requirements.txt
       pip install pyinstaller

  4. Run the build again:
       python build.py --onefile

ISSUE #2 — ModuleNotFoundError: No module named 'customtkinter'
────────────────────────────────────────────────────────────────────────────────

Root cause:
  pip installed the package into a different Python environment
  than the one being used to run the app.

Fix:
  Always activate the virtual environment BEFORE running pip or python.
  You should see (.venv) at the start of your terminal prompt.

  rmdir /s /q .venv
  py -m venv .venv
  Windows (CMD):        .venv\Scripts\activate
  Windows (PowerShell): .venv\Scripts\Activate.ps1
  Linux / macOS:        source .venv/bin/activate

  If PowerShell blocks scripts, run this once first:
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

ISSUE #3 — Ollama not using GPU / responses are very slow
────────────────────────────────────────────────────────────────────────────────

Root cause:
  - NVIDIA CUDA drivers not installed, so Ollama falls back to CPU.
  - Model (e.g. llama3.1:8b) is too large for the available VRAM.

Fix:
  1. Install latest NVIDIA drivers from: https://www.nvidia.com/en-us/drivers/
  2. Verify GPU is visible:
       nvidia-smi
  3. Use a model that fits in your VRAM:
       4 GB VRAM  → ollama pull llama3.2:3b
       8 GB VRAM  → ollama pull llama3.1:8b
  4. Update MODEL_NAME in config.json to match the pulled model.

ISSUE #4 — Web UI error: app.js TypeError: this.setupNavigation is not a function
────────────────────────────────────────────────────────────────────────────────

Root cause:
  setupNavigation() was called in App.init() but its body was missing
  from app.js (replaced with a comment placeholder).

Fix:
  Run: git pull
  Then restart Rona. This is fixed in the latest version on GitHub.

────────────────────────────────────────────────────────────────────────────────
ISSUE #5 — Web UI error: 400 BAD REQUEST on /api/analyze_save
────────────────────────────────────────────────────────────────────────────────

Root cause A — No journal data:
  The data/ folder is private and not on GitHub.
  You must first write and save journal entries in the web UI
  before the Analyze button can work.

Root cause B — Date range is backwards (e.g. June → March):
  Start month must come BEFORE end month.
  Example: Start = January, End = June (not the other way around).

Fix:
  Run: git pull
  Then restart Rona. Date validation is fixed in the latest version.
